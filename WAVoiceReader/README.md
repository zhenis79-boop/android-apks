# WA Voice Reader — прототип

Приложение следит за папкой голосовых сообщений WhatsApp, автоматически
отправляет новые файлы в OpenAI Whisper API и показывает текст поверх
экрана, не открывая WhatsApp.

## Способ А (просто, без установки программ): сборка через GitHub Actions

Так вы получаете готовый APK, вообще не устанавливая Android Studio —
всю сборку делает бесплатный сервер GitHub, а вы только скачиваете
готовый файл.

1. Зарегистрируйтесь на https://github.com (если ещё нет аккаунта) —
   бесплатно, только email.
2. Нажмите "+" в правом верхнем углу → "New repository". Назовите
   репозиторий как угодно (например `wa-voice-reader`), поставьте
   галочку "Private" (чтобы код не был публичным) → "Create repository".
3. На странице пустого репозитория нажмите ссылку
   "uploading an existing file".
4. Перетащите мышкой **всю папку `WAVoiceReader` целиком** (или все
   файлы и подпапки из неё) в область загрузки браузера — Chrome и
   большинство современных браузеров сохраняют структуру папок при
   перетаскивании. Внизу нажмите "Commit changes".
5. Откройте вкладку **"Actions"** в верхнем меню репозитория. Там
   автоматически запустится сборка (жёлтый кружок → зелёная галочка
   через 3-5 минут). Если Actions не запустились сами — зайдите во
   вкладку Actions → выберите workflow "Build APK" → "Run workflow".
6. Когда сборка завершится (зелёная галочка), откройте её, пролистайте
   вниз до раздела **"Artifacts"** и скачайте `WAVoiceReader-apk.zip`.
7. Внутри архива — готовый `app-debug.apk`. Перекиньте его на телефон
   и установите как обычно (разрешив установку "из неизвестных
   источников", если система спросит).

Каждый раз, когда я пришлю вам обновлённый код — просто загружайте
изменённые файлы в тот же репозиторий (кнопка "Add file" →
"Upload files"), и через несколько минут во вкладке Actions будет
новый готовый APK.

## Способ Б: Android Studio на компьютере

1. Установите **Android Studio** (последнюю версию) на компьютер.
2. Откройте эту папку (`WAVoiceReader`) через File → Open в Android Studio.
3. Android Studio сама предложит скачать нужную версию Gradle (обёртку
   `gradlew` мы не кладём в архив — Studio создаст её сама при первом
   открытии, просто нажмите "OK"/"Sync now", когда появится запрос).
4. Дождитесь Gradle Sync (может занять пару минут при первом запуске).
5. Build → Build Bundle(s)/APK(s) → Build APK(s).
6. Готовый файл будет в `app/build/outputs/apk/debug/app-debug.apk`.
7. Скиньте apk на телефон (например, через Google Drive/Telegram себе)
   и установите вручную — при установке система спросит разрешение на
   "установку из неизвестных источников", это нормально для sideload.

## Как настроить после установки

1. Откройте приложение.
2. Нажмите "1. Разрешить доступ ко всем файлам" → включите тумблер для
   этого приложения на открывшемся системном экране.
3. Нажмите "2. Разрешить показ поверх экрана" → включите разрешение.
4. Нажмите "3. Разрешить уведомления" (на Android 13+).
5. Вставьте свой OpenAI API-ключ (начинается на `sk-...`) и нажмите
   "Сохранить ключ". Ключ берётся с https://platform.openai.com/api-keys.
6. Нажмите "▶ Запустить слежение". Появится постоянное уведомление —
   это нормально, так Android разрешает работать фоновому сервису.

С этого момента при получении нового голосового сообщения в WhatsApp
(с включённой автозагрузкой медиа) через несколько секунд наверху
экрана появится всплывающая карточка с текстом.

## Тестовая кнопка без WhatsApp

На главном экране есть кнопка **"🎤 Тестовое голосовое (эмуляция WhatsApp)"**.
Она:

1. Запрашивает разрешение на микрофон (один раз).
2. Пишет 3 секунды звука с микрофона в файл `TEST_<дата>_<время>.opus`.
3. Кладёт этот файл **в отдельную тестовую папку**
   `Android/media (нет) → /WAVoiceReaderTest/VoiceNotes/` на карте памяти —
   она не имеет отношения к WhatsApp и создаётся сама при первом запуске,
   даже если WhatsApp на телефоне вообще не установлен.

Сервис `VoiceWatcherService` слушает сразу **несколько папок одновременно**:
все найденные настоящие папки WhatsApp (если он установлен) и эту тестовую
папку — параллельно. Поэтому:
- без WhatsApp тестовая кнопка всё равно проверяет всю цепочку целиком
  (обнаружение файла → распознавание → показ оверлея), не притворяясь,
  что это папка WhatsApp;
- при реальном использовании настоящие голосовые из WhatsApp по-прежнему
  подхватываются как обычно.

**Порядок теста:** сначала нажмите "▶ Запустить слежение", подождите пару
секунд, затем нажмите тестовую кнопку и скажите что-нибудь вслух. Через
несколько секунд должна появиться карточка с текстом.

На Android 9 и старее нет кодека Opus для записи (доступен с Android 10) —
на таких версиях тест запишется в формате AMR с расширением `.opus` (файл
нестандартный, но наблюдатель его всё равно поймает; ответ от Whisper может
быть менее точным). Для полноценной проверки лучше использовать Android 10+.

## Важные ограничения этой версии (прототип)

- Whisper API — платный (доли цента за минуту аудио), нужен интернет.
- **Не различает входящие и исходящие голосовые.** WhatsApp хранит все голосовые
  сообщения (и полученные, и отправленные вами) в одной папке с одинаковым
  форматом имени файла (`PTT-ГГГГММДД-WAxxxx.opus`) — по самому файлу понять
  направление невозможно. Сейчас программа транскрибирует всё подряд. Настоящий
  фикс потребует `NotificationListenerService` (сопоставление нового файла с
  уведомлением WhatsApp по времени) — отдельная, более объёмная задача,
  сознательно отложенная на потом.
- Не показывает, от какого контакта пришло сообщение — только текст.
  Это можно добавить следующим шагом через NotificationListenerService
  (чтение уведомлений WhatsApp), но это отдельная, более сложная задача.
- В WhatsApp должна быть включена автозагрузка голосовых сообщений
  (обычно включена по умолчанию).
- Некоторые прошивки (MIUI, ColorOS и т.п.) агрессивно убивают фоновые
  сервисы — может понадобиться вручную отключить "оптимизацию батареи"
  для приложения в настройках телефона.
- Путь к папке голосовых зависит от версии WhatsApp/Android — в коде
  (`VoiceWatcherService.CANDIDATE_DIRS`) перечислены 3 самых частых
  варианта. Если не находит — пришлите реальный путь, поправим.

## Структура проекта

- `MainActivity.kt` — экран настроек и разрешений.
- `VoiceWatcherService.kt` — фоновый сервис, следит за папкой (FileObserver
  + резервный опрос раз в 8 сек), запускает распознавание.
- `WhisperClient.kt` — отправка файла в OpenAI Whisper API.
- `OverlayController.kt` — рисует всплывающие карточки с текстом.
- `Prefs.kt` — хранение API-ключа.
- `BootReceiver.kt` — автозапуск сервиса после перезагрузки телефона.
