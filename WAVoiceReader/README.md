# WA Voice Reader — прототип

WA Voice Reader следит за папками голосовых сообщений WhatsApp, отправляет новые
Ogg/Opus-файлы в OpenAI audio transcription API и показывает распознанный текст
поверх экрана, не открывая WhatsApp.

Проект сейчас рассчитан на простой цикл тестирования через GitHub Actions и
Obtainium: код меняется в репозитории, GitHub собирает подписанный release APK,
а телефон получает новую версию из GitHub Releases.

## Рабочий цикл разработки

1. Правки вносятся в код проекта `WAVoiceReader/`.
2. Коммит попадает в ветку `main`.
3. GitHub Actions запускает `.github/workflows/build-apk.yml`.
4. Workflow собирает подписанный `release` APK.
5. APK публикуется в GitHub Releases с версией `v1.0.<github.run_number>`.
6. Obtainium на телефоне видит новый release, скачивает APK и предлагает обновление.
7. После установки приложение тестируется на реальном Android-устройстве.

Локальная сборка на этой машине не требуется. Если сборка не проходит, смотреть
нужно логи GitHub Actions, а не пытаться чинить локальный Android SDK.

## GitHub Actions и релизы

Workflow `Build and Release APK` запускается автоматически при push в `main`, а
также вручную через `workflow_dispatch`.

Для каждого запуска используются:

- `versionCode = github.run_number`;
- `versionName = 1.0.<github.run_number>`;
- tag release: `v1.0.<github.run_number>`;
- имя APK: `WAVoiceReader-v1.0.<github.run_number>.apk`.

Уникальное имя APK важно для Obtainium: так Android и сам Obtainium не путают
новые сборки со старыми файлами в Download.

Для подписанного release APK в настройках репозитория должны быть заданы secrets:

- `RELEASE_KEYSTORE_BASE64`;
- `RELEASE_KEYSTORE_PASSWORD`;
- `RELEASE_KEY_ALIAS`;
- `RELEASE_KEY_PASSWORD`.

Подпись должна оставаться одной и той же между версиями, иначе Android не даст
обновить приложение поверх уже установленного APK.

## Настройка Obtainium

В Obtainium добавьте приложение из GitHub-репозитория:

- repo URL: `https://github.com/zhenis79-boop/android-apks`;
- источник обновлений: GitHub Releases;
- APK asset: `WAVoiceReader-v*.apk`.

После каждого успешного GitHub Actions run появится новый release, и Obtainium
сможет поставить обновление на телефон.

## Как настроить приложение после установки

1. Откройте WA Voice Reader.
2. Нажмите `1. Разрешить доступ ко всем файлам` и включите доступ для приложения.
3. Нажмите `2. Разрешить показ поверх экрана` и включите overlay-разрешение.
4. Нажмите `3. Разрешить уведомления` на Android 13+.
5. Нажмите `Доступ к уведомлениям` и включите доступ для WA Voice Reader.
6. Вставьте OpenAI API-ключ и нажмите `Сохранить ключ`.
7. Нажмите `Запустить слежение`.

Постоянное уведомление после запуска сервиса — нормальное поведение Android для
фоновой работы.

## Тест без WhatsApp

На главном экране есть кнопка `Тестовое голосовое (эмуляция WhatsApp)`.
Она записывает 3 секунды звука с микрофона и кладёт файл в тестовую папку:

`/storage/emulated/0/WAVoiceReaderTest/VoiceNotes`

Эту папку слушает тот же `VoiceWatcherService`, поэтому тест проверяет всю цепочку:
обнаружение файла, отправку в OpenAI, распознавание, историю и overlay-карточку.

Порядок теста:

1. Запустите слежение.
2. Подождите пару секунд.
3. Нажмите тестовую кнопку и скажите короткую фразу.
4. Дождитесь всплывающей карточки с текстом.
5. Если что-то пошло не так, скопируйте логи из блока `Логи` в приложении.

## Что приложение умеет сейчас

- Следит за реальными папками WhatsApp и WhatsApp Business, если они есть на телефоне.
- Дополнительно слушает тестовую папку без WhatsApp.
- Использует `FileObserver` и резервный polling, чтобы не пропускать файлы.
- Следит только за текущим и предыдущим месяцем голосовых, чтобы не сканировать годы истории.
- Читает уведомления WhatsApp через `NotificationListenerService`, чтобы определить отправителя.
- Если уведомление не сопоставилось, всё равно распознаёт голосовое без имени отправителя.
- Хранит последние распознавания в истории.
- Показывает диагностические логи прямо в UI, чтобы удобно присылать их после тестов.

## Ограничения

- Нужен интернет и OpenAI API-ключ.
- Распознавание через OpenAI API платное.
- Если у WhatsApp выключена автозагрузка голосовых, файл может не появиться вовремя.
- Если доступ к уведомлениям завис на уровне прошивки, нужно открыть экран доступа к уведомлениям
  из приложения, выключить тумблер WA Voice Reader и включить обратно.
- На MIUI, ColorOS и похожих прошивках может понадобиться отключить оптимизацию батареи для приложения.
- Если WhatsApp хранит голосовые в другом пути, нужно добавить этот путь в
  `VoiceWatcherService.CANDIDATE_DIRS`.
- При отсутствии уведомления приложение может иногда распознать исходящее голосовое как обычное
  `Голосовое сообщение`; это осознанный fallback, чтобы не терять реальные входящие.

## Структура проекта

- `MainActivity.kt` — экран разрешений, API-ключа, теста, истории и логов.
- `VoiceWatcherService.kt` — foreground service, наблюдение за файлами, очередь распознавания.
- `WaNotificationListener.kt` — чтение уведомлений WhatsApp.
- `SenderTracker.kt` — сопоставление уведомления и файла по времени.
- `WhisperClient.kt` — отправка аудио в OpenAI transcription API.
- `OverlayController.kt` — всплывающие карточки поверх экрана.
- `HistoryStore.kt` — история распознанных сообщений.
- `Logger.kt` — файловые логи для диагностики из UI.
- `Prefs.kt` — настройки приложения.
- `BootReceiver.kt` — автозапуск сервиса после перезагрузки, если слежение было включено.
